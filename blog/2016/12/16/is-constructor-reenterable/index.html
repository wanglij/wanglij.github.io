Is Constructor Reenterable?

Dec 16th, 2016 | Comments

Background

While I was working on an issue in my work, I noticed another weird unexpected issue. The program is multi-threaded program written in C/C++, which runs in QNX platform. There is a singleton class in that program that creates a new thread at the end of its constructor method. Since it is a singleton, so we assume that there should be one and only one thread created, but as a matter fact, most of the time there were two of them running in the process. Eventually, I found that the static function getting the instance of the singleton, get_instance(), was called again within the function executed by the new thread. At that point, the parent thread may not return from the constructor yet, so by calling get_instance() in the new thread the constructor method may be executed the second time, and consequently created another new thread.
