VNC Over SSH Tunnel

Jan 14th, 2016 | Comments

Precondition:

TigerVNC was installed and running in my Arch Linux virtual box. I have TightVNC viewer installed in my local Windows computer. PuTTY is installed in the Windows computer.

Issue:

I could be able to connect to my VNC server directly before, but recent it seems like the port number was blocked by my company network, so I got the following error whenever I tried to connect to my Linux VNC server.

—————————
192.168.91.100::5901 - TightVNC Viewer
—————————
Error in TightVNC Viewer: An attempt was made to access a socket in a way forbidden by its access permissions.
—————————
OK
—————————

Solution: connect VNC over SSH tunnel

On the server (optional)

Edit /etc/systemd/system/vncserver@:1.service, and change the following line ExecStart=/usr/bin/vncserver -geometry 1920x1080 -fg -alwaysshared -dpi 100 %i to ExecStart=/usr/bin/vncserver -geometry 1920x1080 -fg -alwaysshared -dpi 100 -localhost %i By doing this change, it will only allow connections from the localhost. This is why it is optional.

On the windows client

Run PuTTY. - Fill “Host Name (or IP address):” field with “@”. Here the host is the IP address of my Linux machine where the VNC server is running. In my case, it’s 192.168.91.100.
- Make sure “SSH” is selected in “Connection Type:”.
Go to the “Connection” -> “SSH” -> “Tunnels” screen.
Under “Ad new forwarded port:”, - Enter 5901 for the “Source port” field. - Enter “127.0.0.1:5901”. - Click the “Add” button. (You should se a line in the text box that reads “L5901 127.0.0.1:5901”) Go back to the “Session” screen, enter a name for the session and save it. Now open the connection, and enter the password when prompted.

Run TightVNC. Change the “Remote Host:” field to 127.0.0.1::5901, and connect to the VNC like before.

  2. How to tunnel Internet traffic over SSH in Windows
